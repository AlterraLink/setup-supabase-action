# Contributing to Setup Supabase Action

Thank you for your interest in contributing! 

## Development

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Test with all package managers
5. Submit a pull request

## Testing

Test your changes by creating a workflow that uses your fork:

```yaml
- uses: YOUR-USERNAME/setup-supabase-action@your-branch
  with:
    package-manager: pnpm
```

## Pull Request Guidelines

- Keep changes focused and atomic
- Update documentation for any new features
- Add tests for new functionality
- Update CHANGELOG.md

## Questions?

Open an issue or discussion if you need help!
